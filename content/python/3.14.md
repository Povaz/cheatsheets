## [card whats-new] What's new in 3.14 {accent: "#c1440e"}

| pep | feature                       | desc                                    | detail                                                                                             |
|-----|-------------------------------|-----------------------------------------|----------------------------------------------------------------------------------------------------|
| 649 | deferred annotations          | default now; annotations evaluated lazily | Use `annotationlib.get_annotations(obj, format=Format.STRING/FORWARDREF/VALUE)`. No runtime cost at `def` time. |
| 750 | template strings              | `t"..."` returns `Template`, not `str`  | Iterate to access static + `Interpolation` parts. Enables safe HTML/SQL/shell escaping in libraries. |
| 758 | `except` without parens       | `except A, B:` is valid                 | Only without an `as` clause. With `as`, parentheses still required.                                |
| 765 | `return`/`break`/`continue` in `finally` | now emits `SyntaxWarning`     | Silently swallows in-flight exceptions — don't do it.                                              |
| 779 | free-threaded Python official | no-GIL build graduated to supported     | Still opt-in build. ~5–10% single-threaded perf penalty.                                           |
| 784 | `compression.zstd`            | zstandard in the stdlib                 | Also integrated with `tarfile` / `zipfile` / `shutil`. Unified `compression.*` namespace.          |
| 734 | `concurrent.interpreters`     | subinterpreters in the stdlib           | True multi-core without no-GIL. Pair with `concurrent.futures.InterpreterPoolExecutor`.            |
| 768 | `sys.remote_exec()`           | attach a debugger to a running PID      | Opt-out via `PYTHON_DISABLE_REMOTE_DEBUG` / `-X disable-remote-debug`.                             |

> [tip] REPL has syntax highlighting by default. Disable with `PYTHON_BASIC_REPL=1` or `NO_COLOR=1`.
> [tip] Cycle GC is now incremental: `gc.collect(1)` runs an increment, not a full sweep.

## [card types] Data types & literals

| type        | example                            | desc                                 | detail                                                                                    |
|-------------|------------------------------------|--------------------------------------|-------------------------------------------------------------------------------------------|
| `int`       | `42`, `0xff`, `0b1010`, `1_000_000` | arbitrary precision                 | Underscores allowed anywhere for readability. Division: `/` float, `//` floor, `%` mod.   |
| `float`     | `3.14`, `1e-9`, `float("inf")`     | IEEE 754 double                      | Use `math.isclose()` for near-equality. Never `==` on floats.                             |
| `complex`   | `2+3j`                             | complex number                       | `.real` / `.imag` attributes.                                                             |
| `str`       | `"hi"`, `"""multi"""`              | immutable Unicode                    | `f"{x=}"` debug; `f"{x:>10,.2f}"` spec (width, align, thousands, precision).              |
| `bytes`     | `b"\x00\xff"`                      | immutable bytes                      | `bytes.fromhex("ff")`, `b.hex()`, `b.decode("utf-8")`.                                    |
| `bytearray` | `bytearray(b"abc")`                | mutable bytes                        | Same API as `bytes`, but in-place mutation works.                                         |
| `bool`      | `True`, `False`                    | subclass of `int`                    | `True + True == 2`. Don't write `== True` — use the value directly.                       |
| `list`      | `[1, 2, 3]`                        | mutable, ordered                     | `lst.append`, `lst.extend`, `lst[:]` copies, `lst[::-1]` reverses.                        |
| `tuple`     | `(1, 2)`, `1, 2`                   | immutable, ordered                   | Hashable if all elements are. Use for fixed records and dict keys.                        |
| `dict`      | `{"a": 1}`                         | ordered since 3.7                    | `d \| other` merge (3.9+); `d.get(k, default)`; `d.setdefault(k, [])`.                    |
| `set`       | `{1, 2}`, `set()`                  | mutable, unordered, unique           | `{}` is an empty **dict**, not a set. Ops: `\|`, `&`, `-`, `^`.                           |
| `frozenset` | `frozenset([1, 2])`                | hashable set                         | Can be a dict key or set member.                                                          |
| `None`      | `None`                             | singleton null                       | Test with `is None`. In 3.14: `operator.is_none` / `is_not_none` as callables.            |
| `t"..."`    | `t"hello {name}"`                  | `Template` object (3.14, PEP 750)    | Returns `Template`, not `str`. `list(t)` → `[str, Interpolation, str, ...]`.              |

> [warn] Mutable default args are evaluated once at `def` time. Use `x=None` and `if x is None: x = []`.

## [card control] Control flow

| construct             | example                                   | desc                       | detail                                                                                        |
|-----------------------|-------------------------------------------|----------------------------|-----------------------------------------------------------------------------------------------|
| `if/elif/else`        | `if x: ...`                                | chained conditional        | Prefer truthiness: `if items:` over `if len(items) > 0:`.                                     |
| `match/case`          | `match p:\n  case (0, 0): "origin"`       | structural pattern match   | 3.10+. Class patterns `case Point(x=0)`, `\|` alternatives, `_` wildcard, `case [h, *t]:`.    |
| `for`                 | `for x in items:`                          | iterate any iterable       | `for i, x in enumerate(items):`; `for a, b in zip(xs, ys, strict=True):`.                     |
| `while`               | `while cond:`                              | loop until false           | `while (chunk := f.read(4096)):` walrus + read.                                               |
| `break` / `continue`  | inside loop                                | exit / skip iteration      | 3.14: in `finally` → `SyntaxWarning` (PEP 765).                                               |
| `try/except`          | `try: ... except A, B: ...`                | handle errors              | 3.14: parens optional with no `as` clause (PEP 758).                                          |
| `try/except*`         | `except* KeyError as eg:`                  | handle `ExceptionGroup`    | Each matching exception handled separately; preserves the group.                              |
| `else` (loop / try)   | `for ...: else:` / `try: ...: else:`       | runs if no break / no exc  | Counter-intuitive — read the keyword as "no exception" / "no break".                          |
| `finally`             | `try: ... finally: cleanup()`              | always runs                | Don't `return` / `break` / `continue` here (PEP 765). Runs even on exception.                 |
| `with`                | `with open(p) as f:`                       | context manager            | Multi: `with A() as a, B() as b:`. Nest via `contextlib.ExitStack`.                           |
| `raise` / `from`      | `raise ValueError("bad") from src`         | with chaining              | `raise ... from None` suppresses the cause; useful when re-raising a translated error.        |
| `assert`              | `assert cond, "msg"`                       | debug-only check           | Stripped under `-O`. Never use for runtime validation / security.                             |

> [warn] `except Exception:` does **not** catch `KeyboardInterrupt` or `SystemExit` — those inherit `BaseException`.

## [card functions] Functions

| form                | example                                 | desc                       | detail                                                                                    |
|---------------------|-----------------------------------------|----------------------------|-------------------------------------------------------------------------------------------|
| `def`               | `def f(x): ...`                          | regular function           | In 3.14 annotations are lazy (PEP 649) — no runtime cost at `def` time.                   |
| default arg         | `def f(x=1): ...`                        | evaluated once at def      | Mutable default bug: `def f(x=[])` — the list is shared across calls.                     |
| `*args`             | `def f(*args): ...`                      | collect extra positional   | `args` is a `tuple`.                                                                      |
| `**kwargs`          | `def f(**kw): ...`                       | collect extra keyword      | `kw` is a `dict`.                                                                         |
| keyword-only        | `def f(*, api_key): ...`                 | after bare `*`             | Forces callers to pass `api_key=` by name.                                                |
| positional-only     | `def f(x, /, y): ...`                    | before `/`                 | 3.8+. `x` cannot be passed by name; useful for `__init__` and stable APIs.                |
| type hints          | `def f(x: int) -> str: ...`              | annotations                | 3.14: lazy by default (PEP 649). Read via `annotationlib.get_annotations(f)`.             |
| `lambda`            | `lambda x: x + 1`                        | anonymous single-expr fn   | No statements. If you need more than one line, use `def`.                                 |
| generator           | `def g(): yield 1`                       | lazy iterator              | `next(g())` pulls one value; `for x in g():` consumes.                                    |
| `yield from`        | `yield from sub()`                       | delegate to sub-iterator   | Also forwards `send` / `throw`.                                                           |
| `async def`         | `async def f(): await x`                 | coroutine function         | Call returns a coroutine — must be `await`ed or scheduled.                                |
| `partial`           | `partial(f, 1, Placeholder, 3)`          | pre-bind args              | 3.14: `functools.Placeholder` holds a positional slot for later.                          |

> [tip] `@functools.cache` is an unbounded `lru_cache(None)` — great for pure-function memoization.

## [card classes] Classes & data

| form                    | example                                | desc                               | detail                                                                                      |
|-------------------------|----------------------------------------|------------------------------------|---------------------------------------------------------------------------------------------|
| `class`                 | `class A(Base): ...`                    | define class                       | Multiple inheritance uses C3 linearization (`A.__mro__`).                                   |
| `__init__`              | `def __init__(self, x):`                | constructor                        | Don't override `__new__` unless customizing allocation (rare).                              |
| `@dataclass`            | `@dataclass class P: x: int`            | auto `__init__`/`__repr__`/`__eq__`| Flags: `frozen=True`, `slots=True`, `kw_only=True`, `order=True`.                           |
| `field(default_factory)`| `x: list = field(default_factory=list)` | mutable defaults, done right       | Works around the mutable-default bug for dataclasses.                                       |
| `@property`             | `@property\ndef x(self): ...`           | computed attribute                 | Setter: `@x.setter def x(self, v): ...`.                                                    |
| `@classmethod`          | `@classmethod def of(cls, x):`          | bound to class                     | Canonical for alternative constructors (`Point.from_polar(...)`).                           |
| `@staticmethod`         | `@staticmethod def util():`             | plain fn in class namespace        | Usually a sign the function should just live at module scope.                               |
| `__repr__`              | `def __repr__(self):`                   | unambiguous debug string           | Aim to round-trip: `eval(repr(obj)) == obj`.                                                |
| `__eq__` + `__hash__`   | pair them                               | equality + hashing                 | Override `__eq__` → must set `__hash__` too (or `None` to make unhashable).                 |
| `__iter__` / `__next__` | `def __iter__(self):`                   | iterable / iterator                | Iterators return `self` from `__iter__`.                                                    |
| `__enter__` / `__exit__`| `with obj:`                             | context manager                    | Return `True` from `__exit__` to swallow the exception.                                     |
| `__slots__`             | `__slots__ = ("x", "y")`                | fixed attribute set                | Saves memory; blocks `__dict__`. `@dataclass(slots=True)` does this for you.                |
| `Protocol`              | `class P(Protocol): def f(self): ...`   | structural typing                  | `typing.Protocol` — duck typing with static-check support.                                  |

> [tip] `dataclass(slots=True, frozen=True, kw_only=True)` is a strong default for value types.

## [card idioms] Comprehensions & idioms

| idiom        | example                                 | desc                          | detail                                                                                   |
|--------------|-----------------------------------------|-------------------------------|------------------------------------------------------------------------------------------|
| list comp    | `[x*2 for x in xs if x > 0]`            | new list                      | Filter clause optional. Prefer comps over `map`/`filter` for readability.                |
| dict comp    | `{k: v for k, v in pairs}`              | new dict                      | Invert: `{v: k for k, v in d.items()}`.                                                  |
| set comp     | `{x.lower() for x in words}`            | new set                       | Deduplicates. Requires hashable elements.                                                |
| gen expr     | `sum(x*x for x in xs)`                  | lazy iterator                 | Parentheses optional when it's the sole function argument.                               |
| f-string     | `f"{price:>10,.2f}"`                    | formatted literal             | `{x=}` → `x=<repr>`. Spec: `[fill][align][sign][#][0][width][,][.precision][type]`.      |
| t-string     | `t"hello {name}"`                       | `Template` object (3.14)      | Not a `str`. Iterate to interleave static text and `Interpolation` values.                |
| walrus `:=`  | `if (n := len(xs)) > 10:`               | assign inside expression      | Also inside comprehensions: `[y for x in xs if (y := f(x))]`.                            |
| ternary      | `a if cond else b`                      | expression form               | Don't nest more than once — reach for `match` or `if/elif` instead.                      |
| unpacking    | `a, *rest = xs`                         | destructure sequences         | `*` collects; `first, *middle, last = xs` works.                                         |
| dict merge   | `{**a, **b}` or `a \| b`                | combine dicts                 | `\|` is 3.9+. Later keys win.                                                            |
| `enumerate`  | `for i, x in enumerate(xs, start=1):`   | index + value                 | Prefer over `range(len(xs))`.                                                            |
| `zip strict` | `zip(a, b, strict=True)`                | error on length mismatch      | 3.10+. In 3.14, `map(..., strict=True)` works the same way.                              |

## [card async] Async & concurrency

| surface                   | example                                        | desc                               | detail                                                                                           |
|---------------------------|------------------------------------------------|------------------------------------|--------------------------------------------------------------------------------------------------|
| `async def`               | `async def fetch(): ...`                       | coroutine function                 | Calling returns a coroutine object; must be awaited or scheduled.                                |
| `await`                   | `result = await coro`                          | suspend, resume on result          | Only valid inside `async def` / async comprehension.                                             |
| `asyncio.run`             | `asyncio.run(main())`                          | top-level entry                    | Creates + closes a loop. Don't nest it; don't call from inside another coroutine.                |
| `TaskGroup`               | `async with TaskGroup() as tg: tg.create_task(c)` | structured concurrency (3.11+) | Cancels siblings on first exception; raises an `ExceptionGroup`.                                 |
| `asyncio.timeout`         | `async with asyncio.timeout(5): ...`           | time-bounded block (3.11+)         | Preferred over `wait_for`.                                                                        |
| `asyncio.gather`          | `a, b = await gather(x(), y())`                | concurrent awaitables              | Older API; prefer `TaskGroup` for new code. `return_exceptions=True` to collect errors.           |
| `async for`               | `async for x in aiter:`                        | iterate async iterable             | Requires `__aiter__` / `__anext__`.                                                              |
| `async with`              | `async with cm as x:`                          | async context manager              | Requires `__aenter__` / `__aexit__`.                                                             |
| `asyncio.Queue`           | `q = Queue(); await q.put(x); x = await q.get()` | async-safe FIFO                  | Non-blocking: always `await` put/get.                                                            |
| `Runner`                  | `with asyncio.Runner() as r: r.run(main())`    | manage a loop manually             | Use when `asyncio.run` is too coarse.                                                            |
| introspection             | `python -m asyncio ps PID` / `pstree PID`      | live tasks (3.14)                  | Also `asyncio.capture_call_graph()` / `print_call_graph()` in-process.                           |
| `InterpreterPoolExecutor` | `concurrent.futures.InterpreterPoolExecutor()` | subinterpreter pool (3.14)         | True multi-core without no-GIL. Objects cross boundaries via pickling or shared memory.          |

> [warn] `time.sleep()` inside `async def` blocks the entire event loop — always `await asyncio.sleep()`.
> [tip] `asyncio.to_thread(sync_fn, *args)` wraps blocking I/O in a thread without leaving async code.
