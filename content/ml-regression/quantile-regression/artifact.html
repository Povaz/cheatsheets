Regression: predict a number

Start from the ground floor: what regression is, before any loss or tree.

  • Supervised learning — learn a function f(x) ≈ y from examples: a feature matrix X and known targets y.
  • Regression — the target y is a continuous number (a price, a demand, a latency), not a category.
  • The question regression answers by default: "given x, what is the single best number?"

"Best" is not self-evident — it is defined by a loss function. That choice is the whole story of this page, and it is the next section.

Loss functions & the conditional mean

A loss function measures how wrong a prediction ŷ is versus the truth y. Training = choose the model that minimises the average loss over the data.

Squared error (L2 / MSE)

The default loss for regression. Errors are penalised quadratically — a miss of 4 hurts 4× as much as a miss of 2.

L2 / mean squared error · per example L₂(y, ŷ) = ½ · (y − ŷ)²

One fact drives everything below:

Key fact

The single number that minimises mean squared error is the mean. So L2 regression estimates the conditional mean E[y | x] — the average y for inputs like x.

This is LightGBM's out-of-the-box behaviour: its default objective is regression — i.e. L2 (aliases: l2, mse, mean_squared_error). Two derivatives we will reuse in §3 and §8:

Derivatives of L2 w.r.t. the prediction ŷ gradient  g = ŷ − y     hessian  h = 1
Mental model

Pick the loss → pick the statistic you estimate. Squared error → the mean. Absolute error → the median. Pinball loss → any quantile (§7). The model machinery barely changes; the loss does.

How LightGBM works, classically

The standard L2 pipeline. Understand this and the quantile version (§8) is one small swap.

LightGBM is a Gradient Boosting Decision Tree (GBDT) framework. It builds an ensemble of trees one at a time (additive training): keep what you have learned, add one new tree that corrects the current errors.

Crucially, each new tree is not fit to y directly. It is fit to the gradient (and curvature) of the loss at the current predictions. Using a second-order Taylor expansion, every data point contributes a first derivative gᵢ and a second derivative hᵢ:

stateCurrent ensemble
F(x)
→
step 1Compute gᵢ, hᵢ of the loss at F(xᵢ)
→
step 2Grow a tree — leaf-wise, histogram splits
→
step 3Leaf value
w* = −ΣG/(ΣH+λ)
→
step 4Update
F ← F + η·tree
⟲ repeat for n_estimators rounds

The pieces that matter here:

  • Second-order leaf values. The optimal value of a leaf is w* = −G/(H+λ), where G, H sum the gradients/hessians of the points in that leaf and λ is L2 leaf regularisation. The same solver works for any loss that supplies g and h.
  • Leaf-wise (best-first) growth. LightGBM splits the leaf with the largest loss reduction, not whole levels — controlled by num_leaves, bounded by max_depth.
  • Histogram splits. Continuous features are bucketed into discrete bins, making split-finding fast.
The hinge

Everything after split-finding depends on the loss only through g and h. Swap the loss → swap the target statistic, reusing the same trees. That is exactly how we get from "mean" to "quantile."

For L2 specifically: g = ŷ − y and h = 1, so a leaf's value −ΣG/(ΣH+λ) is essentially the mean residual in that leaf — and the ensemble converges toward the conditional mean. Hold that thought; §8 breaks it on purpose.

Why one number isn't enough

The conditional mean answers "what is the typical y?" — a single point. It says nothing about spread or uncertainty, and the real questions often live there:

  • "Give me a range I can be 90% confident in."
  • "What is the worst-case demand I should stock for?"
  • "What 95th-percentile latency do I promise in the SLA?"

The mean is also fragile: skew and outliers drag it. scikit-learn's own example notes that when noise is skewed toward large values, the predicted median sits below the predicted mean, and the median is smoother thanks to its robustness to outliers.

So

To answer range / worst-case / percentile questions we need conditional quantiles, not just the conditional mean. The next two sections build "quantile" precisely.

Quantiles

The α-quantile of a distribution is the value below which a fraction α of the outcomes fall, for α ∈ (0, 1).

αNameMeaning
0.5medianhalf the outcomes fall below it
0.990th percentileonly 10% of outcomes exceed it
0.9595th percentileonly 5% exceed it (a typical "upper bound")
0.055th percentileonly 5% fall below it (a typical "lower bound")

A conditional quantile qα(x) is the α-quantile of y given x — e.g. "the 95th-percentile delivery time for orders that look like this one."

Quantile regression

Quantile regression estimates the median or other quantiles of y conditional on X, while ordinary least squares (OLS) estimates the conditional mean. (scikit-learn, verbatim.)

In other words: instead of one model for the average, you build a model whose output is the conditional α-quantile qα(x).

  • One model targets one α. Want the 5th, 50th and 95th percentiles? Fit three models (see §9).
  • Robust to outliers — the loss is linear in the residual, not squared, so extreme values pull less.
  • Origin — popularised by Koenker & Hallock, "Quantile Regression", Journal of Economic Perspectives 15(4), 2001.

What makes a model estimate a quantile instead of a mean? A different loss — the pinball loss, next.

The pinball (quantile) loss

The loss whose minimiser is the α-quantile. Also called the quantile loss or check function.

Let the residual be t = y − ŷ. The pinball loss for target quantile α is:

Pinball loss · per example (scikit-learn) PBα(t) = α · max(t, 0) + (1 − α) · max(−t, 0)

  =  α · t    if t > 0  (under-predicted: y > ŷ)
    (α − 1) · t  if t < 0  (over-predicted:  y < ŷ)
    0          if t = 0

It is an asymmetric V. The two arms have different slopes, and α sets the tilt:

  • High α → under-predicting is expensive → predictions pushed up → lands on a high quantile.
  • α = 0.5 → symmetric V → this is exactly the mean absolute error, whose minimiser is the median.
  • The minimiser of expected pinball loss is the α-quantile. That single property is the entire trick.
The dial

Pinball loss is one knob: α sets the slope asymmetry, which sets which quantile the optimum lands on. Nothing else needs to change conceptually.

Quantile loss inside LightGBM

The centrepiece. We plug the pinball loss into the §3 machinery — and discover it doesn't fit cleanly. Here is the twist that makes it work.

The gradient and hessian of pinball loss

Recall (§3) that LightGBM only needs g and h. For its quantile objective, differentiating the pinball loss gives (verbatim from LightGBM source):

LightGBM quantile objective · per example gradient  g = (1 − α)  if ŷ ≥ y,   else  −α
hessian   h = 1

Two things are wrong compared to the well-behaved L2 case:

  • The gradient is piecewise-constant. It only encodes a direction — "am I above or below the point?" — scaled by α vs 1−α. It carries no magnitude.
  • The hessian is fake. Pinball loss is piecewise-linear, so its true second derivative is 0 everywhere (undefined at the kink). LightGBM sets h = 1 as a placeholder just to keep the solver running.
Why this breaks the leaf formula

With a constant gradient and a placeholder h = 1, the Newton leaf value w* = −ΣG/(ΣH+λ) is just a fixed nudge per leaf. It knows the direction to move but not how far to move to actually reach the α-quantile. Plain Newton boosting on pinball loss cannot calibrate the quantile.

LightGBM's fix: renew the leaf outputs

LightGBM special-cases the quantile objective. Its source marks IsRenewTreeOutput() = true: after the tree structure is chosen, every leaf's value is overwritten. The replacement is the (weighted) α-percentile of the residuals (y − current prediction) of the samples that landed in that leaf — computed via LightGBM's WeightedPercentileFun(α).

step 1Pinball gradient ± → directions only
→
step 2Grow tree: gradient signs pick where to split
→
renew ★Each leaf ← α-percentile of its residuals (what to predict)
→
step 4Update F ← F + η·tree

So the labour is split: the gradient decides the tree's shape; the percentile sets each leaf's value. The result is an ensemble that estimates the conditional α-quantile.

Using it in LightGBM

Only two knobs separate a quantile model from the default regressor:

ParameterMeaningDefault
objectiveset to "quantile" to train with pinball loss"regression" (L2)
alphathe target quantile α ∈ (0, 1); also used by huber0.9

Fit one model per quantile. To get a lower bound, a center, and an upper bound, train three:

python · one LightGBM model per quantile
from lightgbm import LGBMRegressor

quantiles = [0.05, 0.50, 0.95]
models = {}
for a in quantiles:
    models[a] = LGBMRegressor(
        objective="quantile",   # pinball loss
        alpha=a,                  # target quantile
        n_estimators=300,
        learning_rate=0.05,
    ).fit(X_train, y_train)

lower  = models[0.05].predict(X_test)   # 5th percentile
median = models[0.50].predict(X_test)   # 50th (robust center)
upper  = models[0.95].predict(X_test)   # 95th percentile
# [lower, upper] is a 90% prediction interval  (95% − 5% = 90%)
Prediction intervals

Per scikit-learn's example: models fit at α = 0.05 and α = 0.95 produce a 90% coverage interval (95% − 5% = 90%). The α = 0.5 model gives the median — on average as many observations above as below.

Evaluate each model with the pinball loss for its own α — in LightGBM via metric="quantile", or in scikit-learn via mean_pinball_loss(y_true, y_pred, alpha=a).

Gotchas & good practice

Quantile crossing

Independently-fit models can cross: at some x the predicted q₀.₉₅ may fall below q₀.₀₅. This is not prevented by default. Sort/clip the predictions, or use a joint/monotone-constrained approach if monotonic ordering is required.

alpha defaults to 0.9

Not 0.5. If you want the median you must set alpha=0.5 explicitly — leaving it default silently trains the 90th percentile.

Evaluate with the right metric

Judge a quantile model with the pinball loss at its own α, never with RMSE — RMSE rewards predicting the mean, which is the wrong target here.

  • One α per model. Many quantiles = many models = more training and serving cost.
  • Robust but fiddlier to tune. The linear loss resists outliers, but the placeholder hessian (§8) makes quantile training more sensitive to learning_rate / n_estimators than plain L2 — budget for tuning.
  • α = 0.5 is the median, not the mean. Use it when you want a robust center; use L2 when you genuinely want the average.

Glossary

  • GBDT — gradient boosting decision trees; ensemble built additively, one tree per round.
  • Objective — the loss LightGBM minimises; it determines the target statistic.
  • Quantile / percentile — value below which a fraction α of outcomes fall.
  • Pinball loss — asymmetric loss whose minimiser is the α-quantile; α=0.5 ≡ MAE.
  • Hessian — second derivative of the loss; 1 for L2, a placeholder for quantile.
  • RenewTreeOutput — LightGBM's step that overwrites quantile leaf values with the α-percentile of residuals.
  • Prediction interval — a [lower, upper] range from two quantile models, e.g. [q₀.₀₅, q₀.₉₅].
