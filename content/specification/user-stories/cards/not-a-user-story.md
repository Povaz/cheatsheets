## [table not-a-user-story] Not a User Story

| code | desc | detail |
|------|------|--------|
| `Task` | *"Add /api/v1/users POST endpoint"* | Belongs as a Task. The story is *"As an admin, I can invite a new user, so they can start using the platform."* |
| `Impl note` | *"Use Redis for session storage"* | Belongs in a spike or ADR, not on the story card. |
| `DoD` | *"code reviewed"*, *"deployed to staging"* | Definition of Done is per-team, applies to every story — not authored per story. |
| `Improvement` | *"a script that resets the test DB"* | Belongs as a Task or Improvement Item. If the only beneficiary is the team, the story format is a contortion (see *Fake Story*). |
